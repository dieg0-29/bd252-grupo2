# 11.2.5. Módulo 5: Ventas

## **Especificación de Procesos Batch: PL/pgSQL**

---

### 🧾**1. Cierre diario de caja**

- **Frecuencia:** Diario, al momento de cerrar la tienda

*Nota:* Este proceso podría ser automático si se define una hora exacta para el cierre de caja, o bien el vendedor podría activarlo a través de la interfaz de la aplicación con el botón "Cerrar Caja"

En caso de activación automática, se usaría la extensión pg_cron que permitiría programar la ejecución del proceso

**Programación:**

```sql
CREATE EXTENSION IF NOT EXISTS pg_cron;

SELECT cron.schedule(
  'cierre_caja',
  '10 21 * * *',
  $$SELECT batch_cerrar_caja(current_date);$$
);
```

**Proceso Batch:**

```sql
CREATE OR REPLACE FUNCTION batch_cerrar_caja(p_dia date)
RETURNS void LANGUAGE plpgsql AS $$
DECLARE
  v_ingresos  numeric(14,2) := 0;
  v_devs      numeric(14,2) := 0;
  v_cpos      numeric(14,2) := 0;
  v_cneg      numeric(14,2) := 0;
BEGIN
  -- pagos cobrados ese día
  SELECT COALESCE(SUM(p.monto_pago),0)
    INTO v_ingresos
  FROM pago p
  JOIN estado_pago ep ON ep.cod_estado_pago = p.cod_estado_pago
  JOIN caja c         ON c.cod_caja = p.cod_caja
  WHERE c.fecha_hora_apertura::date = p_dia
    AND lower(ep.nombre_estado_pago) = 'pagado';

  -- devoluciones (egreso)
  SELECT COALESCE(SUM(d.monto_devolucion),0)
    INTO v_devs
  FROM devolucion d
  JOIN caja c ON c.cod_caja = d.cod_caja
  WHERE c.fecha_hora_apertura::date = p_dia;

  -- cambios (ingreso/egreso)
  SELECT
    COALESCE(SUM(CASE WHEN cp.diferencia_cambio>0 THEN cp.diferencia_cambio END),0),
    COALESCE(SUM(CASE WHEN cp.diferencia_cambio<0 THEN -cp.diferencia_cambio END),0)
  INTO v_cpos, v_cneg
  FROM cambio_producto cp
  JOIN caja c ON c.cod_caja = cp.cod_caja
  WHERE c.fecha_hora_apertura::date = p_dia;

  -- actualizar caja del día
  UPDATE caja
  SET monto_total_ingresos = ROUND(v_ingresos - v_devs + v_cpos - v_cneg, 2),
      monto_cierre        = ROUND(monto_apertura + (v_ingresos - v_devs + v_cpos - v_cneg), 2)
  WHERE fecha_hora_apertura::date = p_dia;
END $$;
```

### 💳**2. Generación de cuotas restantes**

- **Frecuencia:** Al momento de registrar una venta a crédito

```sql
CREATE OR REPLACE FUNCTION batch_generar_cuotas_faltantes()
RETURNS int LANGUAGE plpgsql AS $$
DECLARE
  r record; v_gen int := 0;
BEGIN
  FOR r IN
    SELECT v.cod_venta
    FROM venta v
    WHERE v.nro_cuotas > 1
      AND EXISTS (SELECT 1 FROM pago p
                  JOIN estado_pago ep ON ep.cod_estado_pago=p.cod_estado_pago
                  WHERE p.cod_venta=v.cod_venta AND p.nro_cuota=1
                    AND lower(ep.nombre_estado_pago)='pagado')
      AND NOT EXISTS (SELECT 1 FROM pago p2 WHERE p2.cod_venta=v.cod_venta AND p2.nro_cuota>=2)
  LOOP
    PERFORM generar_pagos_restantes(r.cod_venta);  -- tu función ya corregida (solo monto_venta)
    v_gen := v_gen + 1;
  END LOOP;
  RETURN v_gen;
END $$;

```

### ⏰**3. Actualización de cuotas vencidas**

- **Frecuencia:** Diario, al cierre de caja

*Nota:* Este proceso debe ser programado al momento del cierre de caja ya que no se podrán recibir más pagos hasta el día siguiente, donde ya habrá vencido el plazo de la deuda. Se programaría al momento que se programe el cierre de caja, o por el contrario el proceso 1 debería llamar a la ejecución de este proceso

**Programación:**

```sql
CREATE EXTENSION IF NOT EXISTS pg_cron;

SELECT cron.schedule(
  'actualizar_morosos',
  '00 21 * * *',
  $$SELECT batch_marcar_vencidos();$$
);
```

**Proceso Batch:**

```sql
CREATE OR REPLACE FUNCTION batch_marcar_vencidos()
RETURNS int LANGUAGE plpgsql AS $$
DECLARE
  v_ep_vencido  int;
  v_ep_pend     int;
  v_rows        int;
BEGIN
  SELECT min(cod_estado_pago) INTO v_ep_vencido FROM estado_pago WHERE lower(nombre_estado_pago)='vencido';
  SELECT min(cod_estado_pago) INTO v_ep_pend    FROM estado_pago WHERE lower(nombre_estado_pago)='pendiente';

  UPDATE pago
  SET cod_estado_pago = v_ep_vencido
  WHERE cod_estado_pago = v_ep_pend
    AND fecha_vencimiento_pago < current_date;

  GET DIAGNOSTICS v_rows = ROW_COUNT;
  RETURN v_rows;
END $$;

```

### ❌**4. Anulación de pagos de ventas anuladas**

- **Frecuencia:** Cuando se anule una venta

```sql
CREATE OR REPLACE FUNCTION batch_anular_pagos()
RETURNS int LANGUAGE plpgsql AS $$
DECLARE
  v_esta_anulada boolean;
  v_af int := 0;
  r record;
BEGIN
  FOR r IN
    SELECT v.cod_venta
    FROM venta v
    JOIN estado_venta ev ON ev.cod_estado_venta=v.cod_estado_venta
    WHERE lower(ev.descp_estado_venta)='anulada'
  LOOP
    v_af := v_af + anular_pagos_por_venta(r.cod_venta); -- tu función previa
  END LOOP;
  RETURN v_af;
END $$;
```

### ♻️**6. Actualización de indicadores y reportes**

- **Frecuencia:** Al momento en que se efectúe el cierre de caja y el proceso 5

```sql
-- Tabla destino sugerida
CREATE TABLE resumen_deudas (dia date, tramo text, cuotas int, saldo numeric(14,2));

CREATE OR REPLACE FUNCTION batch_resumen_deudas(p_dia date DEFAULT current_date)
RETURNS void LANGUAGE plpgsql AS $$
BEGIN
  INSERT INTO resumen_deudas(dia, tramo, cuotas, saldo)
  SELECT p_dia, tramo, COUNT(*), ROUND(SUM(monto_pago),2)
  FROM (
    SELECT
      CASE
        WHEN p.fecha_vencimiento_pago IS NULL                         THEN 'Sin vencimiento'
        WHEN p.fecha_vencimiento_pago >= p_dia                        THEN '0 - al día'
        WHEN p_dia - p.fecha_vencimiento_pago BETWEEN 1 AND 30        THEN '1-30'
        WHEN p_dia - p.fecha_vencimiento_pago BETWEEN 31 AND 60       THEN '31-60'
        WHEN p_dia - p.fecha_vencimiento_pago BETWEEN 61 AND 90       THEN '61-90'
        ELSE '90+'
      END AS tramo,
      p.monto_pago
    FROM pago p
    JOIN estado_pago ep ON ep.cod_estado_pago=p.cod_estado_pago
    WHERE lower(ep.nombre_estado_pago) IN ('pendiente','vencido')
  ) x
  GROUP BY tramo;
END $$;

```
