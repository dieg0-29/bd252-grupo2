## Caso de uso #1: Gestionar Recursos (vehículos y personal)

| **Campo** | **Detalle** |
|---|---|
| **ID** | **R-201** |
| **Actor(es)** | Jefe de Transporte |
| **Objetivo** | Mantener el registro y la disponibilidad de **vehículos** y **operadores** (conductores/personal de carga). |
| **Precondiciones** | El usuario de transporte debe estar logueado en el sistema. |
| **Disparador** | El usuario ingresa al módulo de Transporte y desea registrar o actualizar unidades u operadores, o consultar su disponibilidad. |
| **Flujo Principal** | 1. El usuario accede a la vista de Transporte.<br>2. Selecciona el menú Recursos.<br>3. Elige “Gestionar Vehículos” o “Gestionar Operadores”.<br>4. Para Vehículos, registra o actualiza datos (placa, capacidad, estado, vigencias).<br>5. Para Operadores, registra o actualiza datos (licencias, contacto, habilitaciones).<br>6. Registra la disponibilidad/turnos y mantenimientos.<br>7. Confirma los cambios para que sean guardados. |
| **Postcondiciones** | Catálogos actualizados y **agenda** de disponibilidad registrada. |
| **Excepciones** | 1. Datos inválidos o duplicados (placa/DNI).  <br>  2.Falta de permisos. |
| **Requerimientos especiales** | Validar vigencias (SOAT/licencia). Historial de cambios. |
| **Frecuencia de uso** | Ocasional (altas/bajas) y diaria (consultar disponibilidad). |

## Caso de uso #2: Integrar Pedidos y Stock

| **Campo** | **Detalle** |
|---|---|
| **ID** | **R-202** |
| **Actor(es)** |  Jefe de Transporte  , Jefe de Ventas , Jefe de Abastecimiento |
| **Objetivo** | **Importar** pedidos de Ventas y **clasificar ítems** en *en stock* vs *en proceso de compra*, quedando listos para planificación cuando ingresen a almacén. |
| **Precondiciones** | Pedidos registrados en Ventas; integración habilitada con Inventario/Abastecimiento. |
| **Disparador** | Nuevo pedido confirmado o actualización de disponibilidad. |
| **Flujo Principal** | 1. El usuario accede a la vista de Transporte.<br>2. Selecciona el menú Pedidos.<br>3. Visualiza la bandeja de pedidos y selecciona un pedido.<br>4. Revisa los detalles (cliente, dirección, ventana, ítems).<br>5. Clasifica los ítems en En stock o En compra.<br>6. Marca los ítems En stock como listos para planificar.<br>7. Activa el aviso de ingreso a stock para los ítems En compra.<br>8. Confirma la clasificación para que quede registrada. |
| **Flujos alternativos** | 1. Pedido con dirección incompleta → enviar a corrección. <br>  2.Ventana conflictiva → sugerir nueva fecha/hora. |
| **Postcondiciones** | Pedido con ítems etiquetados; **cola de planificación** actualizada. |
| **Excepciones** | 1. Falla en integración.  <br>  2. Inconsistencia de unidades/UM. |
| **Requerimientos especiales** | Notificación automática al cambiar estado de inventario. |
| **Frecuencia de uso** | Diaria/continua. |

## Caso de uso #3: Planificar Despachos y Rutas

| **Campo** | **Detalle** |
|---|---|
| **ID** | **R-203** |
| **Actor(es)** | Jefe de Transporte |
| **Objetivo** | **Consolidar** pedidos por zona/fecha, **validar capacidad** y **secuenciar paradas** (hoja de ruta). |
| **Precondiciones** | Ítems **en stock** listos para despacho (R-202); recursos con disponibilidad (R-201). |
| **Disparador** | Inicio de planificación del día o nueva demanda prioritaria. |
| **Flujo Principal** | 1. El usuario accede a la vista de Transporte.<br>2. Selecciona el menú Planificación.<br>3. Filtra por fecha/zona/cliente.<br>4. Selecciona pedidos e inicia el despacho.<br>5. Secuencia las paradas (orden sugerido por cercanía/ventanas, editable).<br>6. Valida la capacidad (peso/volumen).<br>7. Guarda el borrador de la hoja de ruta. |
| **Flujos alternativos** | 1. Exceso de capacidad → dividir en 2 despachos. |
| **Postcondiciones** | Despacho(s) **planificado(s)** con paradas y estimados. |
| **Excepciones** | 1.  Falta de datos de peso/volumen. |
| **Requerimientos especiales** | Cálculo simple de distancia/tiempo; soporte multi-obra en un día. |
| **Frecuencia de uso** | Diaria. |

## Caso de uso #4: Asignar y Coordinar Carga

| **Campo** | **Detalle** |
|---|---|
| **ID** | **R-204** |
| **Actor(es)** |  Jefe de Transporte  , Jefe de Abastecimiento , Jefe de Abastecimiento |
| **Objetivo** | **Asignar** vehículo/operador o **tercero**, **reservar bahía/ventana** con Almacén y **emitir lista de picking**. |
| **Precondiciones** | Despacho planificado (R-203); recursos disponibles (R-201). |
| **Disparador** | Confirmación de despacho a ejecutar. |
| **Flujo Principal** | 1. El usuario accede a la vista de Transporte.<br>2. Selecciona el menú Despachos y abre el despacho a asignar.<br>3. Ingresa a Asignación.<br>4. Selecciona vehículo y operador o el transportista tercero.<br>5. Reserva la bahía/ventana de carga con Almacén.<br>6. Envía la lista de picking a Almacén.<br>7. Confirma la asignación para que quede registrada. |
| **Flujos alternativos** | 1. Choque de bahía → elegir nueva ventana.  <br>  2. Recurso ocupado → reemplazar recurso. |
| **Postcondiciones** | Despacho **asignado** con slot de carga confirmado y picking enviado. |
| **Excepciones** | 1. Capacidad insuficiente detectada. |
| **Requerimientos especiales** | Reglas de permisos (tercerizar requiere autorización). |
| **Frecuencia de uso** | Diaria. |

## Caso de uso #5: Monitorear Ejecución en Ruta

| **Campo** | **Detalle** |
|---|---|
| **ID** | **R-205** |
| **Actor(es)** | Jefe de Transporte ,  Cliente (notificado) |
| **Objetivo** | Hacer seguimiento en tiempo operativo a **estados** del despacho y **notificar al cliente** (salida, en camino, entregado). |
| **Precondiciones** | Despacho asignado (R-204). |
| **Disparador** | Inicio de carga/salida a ruta. |
| **Flujo Principal** | 1. El usuario accede a la vista de Transporte.<br>2. Selecciona el menú Tablero del día.<br>3. Visualiza los despachos y paradas en curso.<br>4. Actualiza estados (en carga → en ruta) según avance.<br>5. Revisa el progreso por parada.<br>6. Se envían notificaciones automáticas (salida/en camino).<br>7. El sistema emite alertas por demoras o incumplimientos de ventana. |
| **Flujos alternativos** | 1. Replanificar orden de paradas por tráfico o ventana. |
| **Postcondiciones** | Trazabilidad de tiempos y estados por despacho/parada. |
| **Excepciones** | 1. Sin conectividad del operador (se guarda y sincroniza). |
| **Frecuencia de uso** | Continua durante la jornada. |

## Caso de uso #6: Registrar Entrega y PoD

| **Campo** | **Detalle** |
|---|---|
| **ID** | **R-206** |
| **Actor(es)** | Operador, Cliente, Sistema |
| **Objetivo** | **Registrar entrega** (completa/parcial) y **Prueba de Entrega** (firma/fotos/notas), asociada a **Guía de Remisión**. |
| **Precondiciones** | Despacho en ruta (R-205); checklist de carga completado; guía generada si aplica. |
| **Disparador** | Llegada a la parada. |
| **Flujo Principal** | 1. El usuario accede a la vista de Transporte.<br>2. Selecciona el menú Ruta/Paradas y abre la parada a atender.<br>3. Visualiza los ítems y cantidades del pedido.<br>4. Registra la entrega (completa o parcial).<br>5. Captura la Prueba de Entrega (PoD) (firma/foto).<br>6. Registra los datos de recepción (nombre/documento).<br>7. Actualiza el estado de la parada. |
| **Flujos alternativos** | 1. Guía no emitida o error de numeración → emitir/regularizar antes de confirmar la entrega. <br>  2. Entrega parcial → anotar en guía y sistema (cantidades y motivo). |
| **Postcondiciones** | Guía entregada y/o firmada, Prueba de entrega almacenado; pedido/parada actualizados. |
| **Excepciones** | 1. Cliente rechaza → ver R-207. <br>  2. Documentación inválida/incompleta. |
| **Requerimientos especiales** | Evidencias accesibles (fotos/documentos). |
| **Frecuencia de uso** | Varias veces al día (por parada). |

## Caso de uso #7: Incidencias, Reprogramación y Postentrega

| **Campo** | **Detalle** |
|---|---|
| **ID** | **R-207** |
| **Actor(es)** | Operador, Planificador, Cliente, Almacén, Sistema |
| **Objetivo** | Registrar un faltante y generar una reentrega, recorriendo el flujo estándar de transporte hasta cerrar el caso. |
| **Precondiciones** | Reclamo del cliente confirmando faltante |
| **Disparador** | Recepción de un reclamo del cliente. |
| **Flujo Principal** | 1. El usuario accede a la vista de Transporte. <br>2. Selecciona el menú Reentregas y abre el caso/pedido afectado. <br>3. Registra el faltante (ítems y cantidades) con motivo y evidencia (fotos/notas). <br>4. El sistema genera un Pedido de Transporte de Reentrega (PTR) asociado y lo deja Pendiente de planificación. <br>5. Si hay stock, marca el PTR como Listo para planificar; si no, activa aviso de ingreso a stock. <br>6. Continúa el flujo estándar: Planificar el PTR (R-203) → Asignar y coordinar carga (R-204) → Monitorear en ruta (R-205) → Registrar entrega y PoD (R-206). <br>7. Al confirmar la entrega de reentrega, el sistema cierra el caso y actualiza el pedido a completado. |
| **Postcondiciones** | Caso **cerrado** o pendiente con **reentrega** en cola; todo **trazado**. |
| **Excepciones** | EX-1: Evidencia obligatoria ausente. |
| **Requerimientos especiales** | Bitácora completa; notificación a Ventas/Cliente/Almacén. |
| **Frecuencia de uso** | Ocasional (según incidencias). |

## Caso de uso #8: Transporte Inbound y Tercerizado

| **Campo** | **Detalle** |
|---|---|
| **ID** | **R-208** |
| **Actor(es)** | Planificador de Transporte, Operador, Proveedor, Almacén, Sistema |
| **Objetivo** | Atender dos escenarios: (A) Recogida solicitada por Abastecimiento (proveedor → Almacén/Obra) y (B) Compra ligada a una venta pendiente (activar envío cuando Abastecimiento define la entrega), incluyendo opción tercerizada. |
| **Precondiciones** | Orden de compra/aviso de Abastecimiento; integración con Inventario; disponibilidad de recursos o confirmación del proveedor. |
| **Disparador** | (A) Abastecimiento solicita recogida en proveedor. / (B) Abastecimiento clasifica la compra ligada a una venta: entrega a Almacén, Proveedor → Obra (tercerizado) o Recogida propia → Obra. |
| **Flujo Principal** | 1. El usuario accede a la vista de Transporte. <br>2. Selecciona el menú Inbound/Tercerizado. <br>3. Elige el tipo de atención: (A) Recogida Abastecimiento o (B) Venta pendiente. (A) Recogida Abastecimiento: <br> 4. Registra la solicitud de recogida (proveedor, OC, destino). <br> 5. Planifica el despacho proveedor → destino (Almacén/Obra) y, si llega a Almacén, coordina bahía/ventana. <br> 6. Ejecución y PoD en el destino (Almacén/Obra); confirma recepción y actualiza inventario si aplica. (B) Compra ligada a venta pendiente: <br> 7. Identifica el pedido de venta y los ítems en compra. <br> 8. Si entrega = Almacén → al ingresar a stock, activa flujo estándar de salida (planificar, asignar, entregar) y notifica a Planificación. <br> 9. Si entrega = Proveedor → Obra (tercerizado) → registra datos del transporte del proveedor, notifica al cliente, marca ítems como tercerizados y cierra la logística propia. <br> 10. Si entrega = Recogida propia → Obra → planifica despacho proveedor → Obra y lo atiende como un despacho normal (checklist/PoD), sin pasar por Almacén. <br> 11. Confirma para guardar cambios y trazabilidad. |
| **Postcondiciones** | Ítems recibidos (y stock actualizado), tercerizados o enviados a obra vía despacho propio; pedido de venta activado/cerrado según corresponda; trazabilidad completa. |
| **Requerimientos especiales** | Coherencia con Abastecimiento/Inventario; permisos para tercerizar. |
| **Frecuencia de uso** | Periódica (según compras y ventas pendientes). |
