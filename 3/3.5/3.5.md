# 3.5. Módulo 5: Ventas

Descripcion: 

El módulo de Ventas se encarga de gestionar el proceso de cierre del acuerdo de adquisición de un producto dentro de la ferretería, registrando transacciones, acuerdos y generando documentos legales como comprobantes de pago o reportes de venta. Su propósito principal es automatizar, controlar y registrar las ventas.

---

| Requerimiento | Nombre |
|---------------|--------|
| R-501 | Registrar Venta al Contado |
| R-502 | Registrar Venta a Crédito |
| R-503 | Registrar Pago |
| R-504 | Generar Guía de Remisión |
| R-505 | Devolución de Producto |
| R-506 | Anular Venta |
| R-507 | Cierre de Caja Diario |

---

## **Caso de Uso R-501: Registrar Venta al Contado**

| **ID** | R-501 |
|---|---|
| **Actor(es)** | Vendedor |
| **Descripción** | Registra una venta pagada en el momento |
| **Precondiciones** | 1. Cliente identificado (o venta rápida).<br>2. Productos existentes y disponibles. |
| **Flujo Principal** | 1. El cajero abre una nueva venta.<br>2. Verifica productos y precios.<br>3. Registra medio de pago y monto.<br>4. Emite comprobante de pago.<br>5. Actualiza stock y cierra la venta. |
| **Requerimientos Especiales** | Integración con lectora de tarjetas o código y control de número de comprobante. |
| **Frecuencia de Uso** | Varias veces al día. |

---

## **Caso de Uso R-502: Registrar Venta a Crédito**

| **ID** | R-502 |
|---|---|
| **Actor(es)** | Vendedor |
| **Descripción** | Registra una venta con pago en cuotas |
| **Precondiciones** | 1. Cliente identificado sin deudas pendientes.<br>2. Productos existentes y disponibles. |
| **Flujo Principal** | 1. El cajero abre una nueva venta.<br>2. Verifica productos y precios.<br>3. Define condiciones (plazo/cuotas).<br>4. Emite comprobante de pago.<br>5. Actualiza stock y queda saldo por cobrar. |
| **Requerimientos Especiales** | Alertas de vencimientos y bloqueo si el cliente no paga a tiempo |
| **Frecuencia de Uso** | Menor que contado, pero constante. |

---

## **Caso de Uso R-503: Registrar Pago**

| **ID** | R-503 |
|---|---|
| **Actor(es)** | Vendedor |
| **Descripción** | Permite registrar pagos de ventas al contado o abonos de ventas a crédito. |
| **Precondiciones** | 1. Venta existente.<br>2. Caja abierta. |
| **Flujo Principal** | 1. El vendedor busca la venta.<br>2. Selecciona “Registrar pago”.<br>3. Indica medio y monto.<br>4. Guarda y actualiza el saldo. |
| **Requerimientos Especiales** | Validación para que el total de pagos no supere el total de la venta. |
| **Frecuencia de Uso** | Alta durante el día. |

---

## **Caso de Uso R-504: Generar Guía de Remisión**

| **ID** | R-504 |
|---|---|
| **Actor(es)** | Vendedor |
| **Descripción** | Crea la documentación y pedido de despacho cuando la venta requiere envío. |
| **Precondiciones** | 1. Venta registrada.<br>2. Dirección de entrega confirmada. |
| **Flujo Principal** | 1. El vendedor marca “Requiere envío”.<br>2. Ingresa punto de partida y llegada.<br>3. Genera guía de remisión.<br>4. Notifica a Transporte para programación. |
| **Requerimientos Especiales** | Debe permitir adjuntar indicaciones (ejm: lugar para estacionar) y horarios del cliente. |
| **Frecuencia de Uso** | Según entregas a domicilio u obra. |

---

## **Caso de Uso R-505: Devolución de producto**

| **ID** | R-505 |
|---|---|
| **Actor(es)** | Vendedor, Cajero, Administrador |
| **Descripción** | Registra la devolución de productos y emite la nota de crédito correspondiente. |
| **Precondiciones** | 1. Existe una venta previa.<br>2. El producto está en condiciones aceptadas por la tienda. |
| **Flujo Principal** | 1. El vendedor abre “Devolución”.<br>2. Selecciona venta y productos a devolver.<br>3. El sistema calcula el monto y genera nota de crédito.<br>4. Reingresa stock si aplica.<br>5. Cierra la devolución. |
| **Requerimientos Especiales** | Control de tiempos/motivos y relación con la venta original. |
| **Frecuencia de Uso** | Baja |

---

## **Caso de Uso R-506: Anular Venta**

| **ID** | R-506 |
|---|---|
| **Actor(es)** | Administrador |
| **Descripción** | Permite anular una venta bajo condiciones definidas y revertir sus efectos. |
| **Precondiciones** | 1. Venta en el mismo día o según política.<br>2. Comprobante no enviado a SUNAT o con proceso de baja permitido. |
| **Flujo Principal** | 1. El administrador busca la venta.<br>2. Revisa motivo y confirma anulación.<br>3. Revierte stock y marca comprobante como anulado o dado de baja. |
| **Requerimientos Especiales** | Auditoría completa y restricción por rol (administrador). |
| **Frecuencia de Uso** | Baja |

---

## **Caso de Uso R-507: Cierre de Caja Diario**

| **ID** | R-507 |
|---|---|
| **Actor(es)** | Vendedor |
| **Descripción** | Resume ventas y pagos del día para cuadrar el dinero de caja. |
| **Precondiciones** | 1. Caja abierta con movimientos. |
| **Flujo Principal** | 1. El cajero selecciona “Cierre de caja”.<br>2. El sistema suma ventas por medio de pago.<br>3. Se ingresan montos físicos contados.<br>4. Se muestra diferencia y se confirma el cierre.<br>5. Se genera reporte. |
| **Requerimientos Especiales** | Exportación a PDF/Excel y bloqueo de nuevas ventas hasta reapertura. |
| **Frecuencia de Uso** | Una vez por día por caja. |
