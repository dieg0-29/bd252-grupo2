# 4.1.Diseño Conceptual: Módulo de Ventas
## 💡 Modelo Conceptual: Módulo de Ventas

![Modelo](<Modelo ER (VENTAS).png>)

---

## Entidades:

| Entidad             | Definición semántica                                                                                              |
| ------------------- | ----------------------------------------------------------------------------------------------------------------- |
| **CLIENTE**         | Persona natural o jurídica que realiza compras y a la cual se asocian beneficios, historial y reclamos.           |
| **VENDEDOR**        | Colaborador responsable de registrar ventas y operar caja en turnos (apertura/cierre).                            |
| **PRODUCTO**        | Bien individual del catálogo con precio unitario que puede venderse, cambiarse o devolverse.                      |
| **VENTA**           | Transacción comercial que agrupa ítems (productos), condiciones (tipo, IGV, entrega) y genera pagos/comprobantes. |
| **PAGO**            | Abono de dinero asociado a una venta (total o parcial), con posible vencimiento si es crédito.                    |
| **METODO_PAGO**     | Catálogo de medios de pago disponibles para cobrar (efectivo, tarjeta, transferencia, etc.).                      |
| **COMPROBANTE**     | Documento fiscal emitido por un pago (boleta, factura, nota), con numeración y tipo normado.                      |
| **CAJA**            | Sesión operativa de punto de venta con apertura/cierre, totales y conteos de movimientos.                         |
| **RECLAMO**         | Registro formal de disconformidad del cliente respecto a una venta o producto.                                    |
| **DEVOLUCION**      | Operación de restitución de dinero por productos/ventas, ejecutada usualmente a partir de un reclamo.             |
| **ANULACION**       | Invalidación formal de una venta o parte de ella (y/o de un pago), revirtiendo efectos contables/fiscales.        |
| **CAMBIO_PRODUCTO** | Operación de intercambio: el cliente retorna un producto y recibe otro en su lugar.                               |

## Atributos:

### **CLIENTE**

| Atributo           | Semántica                             | Ontología (razón de ser)                    | Formato                  | Rango / Reglas            |
| ------------------ | ------------------------------------- | ------------------------------------------- | ------------------------ | ------------------------- |
| **cod_cliente**    | Identificador único del cliente.      | Llave para ventas, reclamos y beneficios.   | `VARCHAR(20)` o `INT` | Único, no nulo           |
| **fecha_registro** | Momento de alta del cliente.          | Antigüedad y auditoría.           | `TIMESTAMP`              | ≤ ahora                  |
| **total_compras**  | Total de compras realizadas. | KPI de valor del cliente. | `INT`          | ≥ 0                      |
| **total_puntos**   | Puntos de fidelización acumulados.    | Beneficios/descuentos y campañas.           | `INT`                    | ≥ 0                      |


### **VENDEDOR**

| Atributo          | Semántica                         | Ontología                                         | Formato                  | Rango / Reglas             |
| ----------------- | --------------------------------- | ------------------------------------------------- | ------------------------ | -------------------------- |
| **cod_vendedor**  | Identificador único del vendedor. | Atribución de ventas y responsabilidades de caja. | `VARCHAR(20)` o `INT` | Único, no nulo            |
| **fecha_ingreso** | Fecha de incorporación.           | Gestión de personal y permisos.                   | `DATE`                   | ≤ hoy                     |
| **total_ventas**  | Total de ventas realizadas  | KPI de desempeño.                     | `INT`          | ≥ 0                      |



### **PRODUCTO**

| Atributo                 | Semántica                   | Ontología                                | Formato                  | Rango / Reglas |
| ------------------------ | --------------------------- | ---------------------------------------- | ------------------------ | -------------- |
| **id_producto**          | Identificador del producto. | Llave del catálogo.                      | `VARCHAR(20)` o `BIGINT` | Único         |
| **nombre_producto**      | Denominación comercial.     | Visualización en listados/comprobantes.  | `VARCHAR(120)`           | No vacío      |
| **descripcion_producto** | Descripción ampliada.       | Información para ficha    | `VARCHAR(500)`           | Opcional      |
| **precio_unitario**      | Precio por unidad.          | Base de cálculo de importes e impuestos. | `DECIMAL(12,2)`          | ≥ 0.00        |



### **VENTA**

| Atributo             | Semántica                                        | Ontología                         | Formato         | Rango / Reglas                                                |
| -------------------- | ------------------------------------------------ | --------------------------------- | --------------- | ------------------------------------------------------------- |
| **id_venta**         | Identificador de la venta.                       | Trazabilidad de la transacción.   | `INT`        | Único                                                        |
| **fecha_hora_venta** | Momento de registro de la venta.                 | Ordenamiento, cortes y auditoría. | `DATETIME`     | ≤ ahora                                                      |
| **monto_venta**      | Importe total de la venta (incl. IGV si aplica). | Base para pagos y comprobantes.   | `DECIMAL(12,2)` | ≥ 0; consistente con ítems                                   |
| **tipo_venta**       | Modalidad de pago.                               | Determina si hay vencimientos.    | `ENUM`          | {`CONTADO`,`CREDITO`}                                        |
| **fecha_entrega**    | Compromiso de entrega.                           | Gestión logística y SLA.          | `DATE`          | ≥ fecha_venta; opcional                                      |
| **igv**              | Monto de IGV de la venta.                        | Cumplimiento tributario.          | `DECIMAL(12,2)` | ≥ 0; ≈ 18% del monto                               |
| **estado_venta**     | Estado del ciclo.                                | Control del proceso y reportes.   | `ENUM`          | {`CANCELADA`,`ANULADA`,`POR PAGAR`} |



### **PAGO**

| Atributo              | Semántica                         | Ontología                              | Formato         | Rango / Reglas                             |
| --------------------- | --------------------------------- | -------------------------------------- | --------------- | ------------------------------------------ |
| **num_pago**          | Identificador/secuencia del pago. | Conciliación y trazabilidad del cobro. | `INT`        | Único por venta                |
| **monto_pago**        | Importe abonado.                  | Disminuye saldo de la venta.           | `DECIMAL(12,2)` | > 0; ≤ saldo                              |
| **fecha_pago**        | Momento de cobro.                 | Corte de caja/tesorería.               | `DATETIME`     | ≤ ahora                                   |
| **fecha_vencimiento** | Límite del pago (si es crédito).  | Motor de cobranza/notificaciones.      | `DATE`          | Nulo si CONTADO; ≥ fecha_venta si CREDITO |



### **METODO PAGO**

| Atributo               | Semántica                 | Ontología (razón de ser)        | Formato       | Rango / Reglas                                    |
| ---------------------- | ------------------------- | ------------------------------- | ------------- | ------------------------------------------------- |
| **id_metodo**          | Identificador del método. | Llave del catálogo de cobranza. | `SMALLINT`    | Único                                            |
| **descripcion_metodo** | Nombre del método.        | Reportes.                  | `VARCHAR(60)` | {Efectivo, Tarjeta, Transferencia, QR, Mixto, …} |


### **COMPROBANTE**

| Atributo             | Semántica                   | Ontología (razón de ser)     | Formato       | Rango / Reglas                                       |
| -------------------- | --------------------------- | ---------------------------- | ------------- | ---------------------------------------------------- |
| **tipo_comprobante** | Clase del documento fiscal. | Reglas tributarias/validez.  | `ENUM`        | {`BOLETA`,`FACTURA`,`NOTA_CREDITO`,`NOTA_DEBITO`}   |
| **nro_comprobante**  | Serie y número.             | Control fiscal y validación. | `VARCHAR(20)` | Patrón típico `F001-00012345`; único por tipo/serie |


### **CAJA**

| Atributo                | Semántica                        | Ontología (razón de ser)       | Formato         | Rango / Reglas         |
| ----------------------- | -------------------------------- | ------------------------------ | --------------- | ---------------------- |
| **id_caja**             | Identificador del turno de caja. | Auditar sesión operativa.      | `INT`        | Único                 |
| **fecha_hora_apertura** | Inicio del turno.                | Punto de control de arqueo.    | `DATETIME`     | < cierre              |
| **monto_apertura**      | Fondo inicial.                   | Base de arqueo.                | `DECIMAL(12,2)` | ≥ 0                   |
| **vendedor_apertura**   | Usuario que abre.                | Responsabilidad operativa.     | FK a `VENDEDOR`   | Obligatorio           |
| **fecha_hora_cierre**   | Fin del turno.                   | Cierre y conciliación.         | `DATETIME`     | ≥ apertura            |
| **monto_cierre**        | Valor al cierre.                 | Cuadre final.                  | `DECIMAL(12,2)` | ≥ 0                   |
| **vendedor_cierre**     | Usuario que cierra.              | Responsabilidad operativa.     | FK a `VENDEDOR`   | Obligatorio al cierre |
| **cantidad_ventas**     | Nº de ventas en el turno.        | Métrica operativa (derivable). | `INT`           | ≥ 0                   |
| **monto_ventas_total**  | Total vendido en el turno.       | Métrica/arqueo (derivable).    | `DECIMAL(12,2)` | ≥ 0                   |


### **RECLAMO**

| Atributo               | Semántica                  | Ontología (razón de ser)      | Formato     | Rango / Reglas |
| ---------------------- | -------------------------- | ----------------------------- | ----------- | -------------- |
| **cod_reclamo**        | Identificador del reclamo. | Trazar atención y resolución. | `INT`    | Único         |
| **fecha_hora_reclamo** | Momento de registro.       | SLA y auditoría.              | `DATETIME` | ≤ ahora       |


### **DEVOLUCION**

| Atributo                  | Semántica                       | Ontología (razón de ser)      | Formato         | Rango / Reglas                   |
| ------------------------- | ------------------------------- | ----------------------------- | --------------- | -------------------------------- |
| **cod_devolucion**        | Identificador de la devolución. | Ajuste contable/caja.         | `INT`        | Único                           |
| **fecha_hora_devolucion** | Momento de la devolución.       | Auditoría de movimiento.      | `DATETIME`     | ≤ ahora                         |
| **motivo_devolucion**     | Causa declarada.                | Análisis de calidad/servicio. | `VARCHAR(200)`  | De acuerdo a catálogo |
| **monto_devuelto**        | Importe restituido.             | Impacto en caja y saldo.      | `DECIMAL(12,2)` | ≥ 0; ≤ monto involucrado.        |


### **ANULACION**

| Atributo                 | Semántica                         | Ontología (razón de ser)                | Formato        | Rango / Reglas                   |
| ------------------------ | --------------------------------- | --------------------------------------- | -------------- | -------------------------------- |
| **cod_anulacion**        | Identificador del acto de anular. | Reverso formal de venta/pago.           | `INT`       | Único                           |
| **fecha_hora_anulacion** | Momento de anulación.             | Auditoría.                              | `DATETIME`    | ≤ ahora                         |
| **motivo_anulacion**     | Causa del reverso.                | Control y prevención de errores/fraude. | `VARCHAR(200)` | De acuerdo a catálogo |


### **CAMBIO PRODUCTO**

| Atributo                  | Semántica                           | Ontología (razón de ser)                        | Formato        | Rango / Reglas                   |
| ------------------------- | ----------------------------------- | ----------------------------------------------- | -------------- | -------------------------------- |
| **cod_cambio**            | Identificador del cambio.           | Trazabilidad del intercambio.                   | `INT`       | Único.                           |
| **fecha_hora_cambio**     | Momento del cambio.                 | Auditoría/stock.                                | `DATETIME`    | ≤ ahora                         |
| **motivo_cambio**         | Causa del intercambio.              | Servicio al cliente / calidad.                  | `VARCHAR(200)` | De acuerdo a acatálogo |
