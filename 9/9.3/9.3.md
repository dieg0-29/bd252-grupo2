# 9.3. Stack Tecnológico Elegido

# 🛠️ Mi Stack de Desarrollo

Este documento describe las tecnologías principales que componen mi stack de desarrollo, cubriendo tanto el **frontend** como el **backend** y las herramientas de **construcción/compilación**.

---

## 🚀 Frontend

El lado del cliente se centra en una interfaz de usuario dinámica, tipada y estilizada eficientemente.

### **React**
* **Tipo:** Librería de JavaScript para construir **interfaces de usuario (UI)**.
* **Breve Descripción:** Permite crear componentes de interfaz reutilizables, gestionando el estado de la aplicación de manera declarativa. Es fundamental para construir **Single Page Applications (SPAs)** con una gran experiencia de usuario.

### **TypeScript**
* **Tipo:** Superconjunto de JavaScript que añade **tipado estático**.
* **Breve Descripción:** Mejora la calidad del código al permitir detectar errores durante la compilación en lugar de la ejecución. Aporta mejor **mantenibilidad, escalabilidad** y una excelente experiencia de desarrollo con autocompletado y refactorización.

### **Tailwind CSS**
* **Tipo:** Framework **CSS Utility-First**.
* **Breve Descripción:** Proporciona clases de utilidad de bajo nivel (como `flex`, `pt-4`, `text-center`) que permiten **construir diseños rápidamente** directamente en el marcado HTML/JSX, sin tener que escribir CSS personalizado tradicional.

---

## ⚙️ Backend y Base de Datos

El lado del servidor se encarga de la lógica de negocio y la persistencia de datos.

### **Node.js**
* **Tipo:** Entorno de ejecución de **JavaScript del lado del servidor**.
* **Breve Descripción:** Permite ejecutar JavaScript fuera del navegador, usando el motor V8 de Chrome. Su naturaleza **asíncrona y basada en eventos** lo hace ideal para construir aplicaciones web escalables y de alto rendimiento (I/O intensivas).

### **Express.js**
* **Tipo:** Framework de **aplicaciones web** minimalista para Node.js.
* **Breve Descripción:** Proporciona un conjunto robusto de características para desarrollar **APIs RESTful** y aplicaciones web. Simplifica la gestión de rutas, *middleware*, y peticiones HTTP en el backend de Node.js.

### **PostgreSQL**
* **Tipo:** Sistema de gestión de **bases de datos relacionales (RDBMS)**.
* **Breve Descripción:** Una base de datos **potente, de código abierto y altamente extensible**. Es conocida por su **fiabilidad, integridad de datos** y soporte para características avanzadas como JSONB, transacciones complejas y consultas eficientes.

---

## 🛠️ Herramientas de Desarrollo

Herramienta esencial para la optimización y velocidad del entorno de desarrollo.

### **Vite**
* **Tipo:** **Tooling** de construcción de próxima generación (*Build Tool*).
* **Breve Descripción:** Ofrece un **servidor de desarrollo local extremadamente rápido** gracias a los módulos ES nativos y una compilación optimizada basada en Rollup para la producción. Es crucial para el **desarrollo rápido** de aplicaciones React/TypeScript.

---

*Este stack combina la velocidad y tipado del desarrollo moderno (React, TypeScript, Vite) con un backend robusto y escalable (Node.js, Express.js, PostgreSQL).*
