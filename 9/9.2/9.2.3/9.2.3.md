# 9.2.3. Módulo 3 / Integrante 3

# Funcionalidad Primaria 

## FLUJO

La funcionalidad primaria elegida es el **Flujo de Recepción y Auditoría de Mercancía (R-302)**. Este proceso traza la auditoría de un producto desde que se asigna la tarea a un operador hasta que el stock se actualiza en el sistema, separando la mercancía "conforme" (vendible) de la "defectuosa" (merma).

### 1. El operador accede al panel de "Gestión de Inventario" y selecciona la tarea de "Recepción".

### 2. El sistema muestra las reservas de recepción asignadas (tareas). El operador selecciona la tarea T-002 (asociada a una orden de compra) y presiona "Recepcionar".

### 3. Se presenta el detalle de los productos esperados. El operador realiza el conteo físico y, al encontrar discrepancias (ej. sacos de cemento dañados), presiona "Reportar Incidencia".

### 4. En la ventana modal, el operador especifica el tipo y la cantidad de la merma (ej. 4 Húmedos, 4 Incompletos, 5 Rotos) y guarda la incidencia.

### 5. Finalmente, el operador ingresa las cantidades conformes (lo bueno) y presiona "Finalizar Recepción". Esta acción dispara la lógica de negocio que actualiza el inventario

## REFLEJA

### El flujo de recepción impacta directamente el Kardex. El "Reporte de Movimientos" (R-308) refleja las entradas generadas.

Los movimientos 'ENTRADA' (stock bueno) y 'MERMA_ENTRADA' (stock defectuoso) generados en el flujo anterior, quedan registrados en el historial del producto (Kardex) para su auditoría, tal como se ve en la pantalla de "Reporte de Movimientos".

*Eventos:*

- Carga de Página:
    
    Se llenará la lista de tareas de recepción asignadas al operador logueado.
    
    ```sql
    -- Cargar tareas de recepción asignadas al usuario actual
    SELECT
      r.cod_reserva,
      r.tipo_reserva,
      c.fecha_cupo,
      t.hora_inicio,
      rec.placa_vehiculo_entrega,
      rec.nombre_conductor_entrega,
      r.estado
    FROM "FERRETERIA".reserva_almacen r
    JOIN "FERRETERIA".operador_reserva_almacen op ON r.cod_reserva = op.cod_reserva
    JOIN "FERRETERIA".cupo_disponible c ON r.cod_cupo = c.cod_cupo
    JOIN "FERRETERIA".turno_almacen t ON c.cod_turno = t.cod_turno
    JOIN "FERRETERIA".recepcion rec ON r.cod_recepcion = rec.cod_recepcion
    WHERE
      op.cod_usuario = ? -- (ID del Operador logueado)
      AND r.tipo_reserva = 'Recepcion'
      AND r.estado IN ('Confirmado', 'En Proceso');
    
    ```
    
- Click en "Operadores Asignados" (abre modal I-0203):
    
    Muestra los compañeros asignados a la misma tarea.
    
    ```sql
    -- Consultar operadores para una reserva específica
    SELECT p.nombre_persona
    FROM "FERRETERIA".operador_reserva_almacen ora
    JOIN "FERRETERIA".usuario u ON ora.cod_usuario = u.cod_usuario
    JOIN "FERRETERIA".persona p ON u.cod_persona = p.cod_persona
    WHERE ora.cod_reserva = ?; -- (ej. 'T-002')
    
    ```

- Click en "Finalizar Recepción":
    
    Ejecuta la transacción principal que actualiza el inventario (Postcondiciones R-302).
    
    ```sql
    -- (INICIO DE TRANSACCIÓN)
    
    -- 1. Actualizar el detalle_recepcion con las cantidades finales
    UPDATE "FERRETERIA".detalle_recepcion
    SET
      cantidad_recibida = ?, -- (Total: conforme + defectuosa)
      cantidad_conforme = ?, -- (Lo bueno, ej. 41)
      cantidad_defectuosa = ? -- (Lo malo, ej. 9)
    WHERE cod_detalle_recepcion = ?; -- (ID del detalle para Cemento)
    
    -- 2. Crear MOVIMIENTO de ENTRADA (stock bueno)
    INSERT INTO "FERRETERIA".movimiento (
      tipo_movimiento,
      cantidad,
      cod_inventario,
      cod_detalle_recepcion
    )
    VALUES (
      'ENTRADA',
      ?, -- (cantidad_conforme, ej. 41)
      (SELECT cod_inventario FROM "FERRETERIA".inventario WHERE cod_producto = ? AND cod_ubicacion = ?), -- (ID de Ubicación de VENTA)
      ? -- (cod_detalle_recepcion)
    );
    -- (Un Trigger en esta tabla debe actualizar "FERRETERIA".inventario.stock_fisico)
    
    -- 3. (Si aplica) Crear MOVIMIENTO de MERMA (stock malo)
    INSERT INTO "FERRETERIA".movimiento (
      tipo_movimiento,
      cantidad,
      cod_inventario,
      cod_detalle_recepcion
    )
    VALUES (
      'MERMA_ENTRADA',
      ?, -- (cantidad_defectuosa, ej. 9)
      (SELECT cod_inventario FROM "FERRETERIA".inventario WHERE cod_producto = ? AND cod_ubicacion = ?), -- (ID de Ubicación de MERMA)
      ? -- (cod_detalle_recepcion)
    );
    -- (Un Trigger en esta tabla debe actualizar "FERRETERIA".inventario.stock_fisico de merma)
    
    -- 4. Actualizar estado de la reserva
    UPDATE "FERRETERIA".reserva_almacen
    SET estado = 'Completado'
    WHERE cod_reserva = ?; -- (ej. 'T-002')
    
    -- (FIN DE TRANSACCIÓN)
    
    ```

- Click en "Buscar":
    
    Muestra el Kardex del producto, reflejando los movimientos generados (Flujo R-308).
    
    ```sql
    -- Consultar el Kardex de un producto (simplificado)
    SELECT
      m.fecha_movimiento,
      m.hora_movimiento,
      m.tipo_movimiento,
      m.cantidad,
      CASE
        WHEN m.cod_detalle_recepcion IS NOT NULL THEN 'Recepción: ' || rec.cod_recepcion::text
        WHEN m.cod_detalle_conteo IS NOT NULL THEN 'Conteo: ' || con.cod_conteo::text
        WHEN m.cod_venta IS NOT NULL THEN 'Venta: ' || v.cod_venta_fmt
        ELSE 'N/A'
      END AS origen_movimiento
    FROM "FERRETERIA".movimiento m
    JOIN "FERRETERIA".inventario i ON m.cod_inventario = i.cod_inventario
    JOIN "FERRETERIA".producto p ON i.cod_producto = p.cod_producto
    LEFT JOIN "FERRETERIA".detalle_recepcion dr ON m.cod_detalle_recepcion = dr.cod_detalle_recepcion
    LEFT JOIN "FERRETERIA".recepcion rec ON dr.cod_recepcion = rec.cod_recepcion
    LEFT JOIN "FERRETERIA".detalle_conteo dc ON m.cod_detalle_conteo = dc.cod_detalle_conteo
    LEFT JOIN "FERRETERIA".conteo con ON dc.cod_conteo = con.cod_conteo
    LEFT JOIN "FERRETERIA".venta v ON m.cod_venta = v.cod_venta
    WHERE
      p.nombre_producto ILIKE ? -- (ej. '%cemento%')
    ORDER BY
      m.fecha_movimiento DESC, m.hora_movimiento DESC;
    
    ```
