# 6.3. Módulo / Integrante 3

# Diseño Físico: Tabla `inventario`

## Estructura lógica (por registro)
Esta tabla guarda el saldo actual de cada producto en su ubicación única.

| Atributo            | Formato (PostgreSQL) | Tamaño                     |
|----------------------|----------------------|-----------------------------|
| cod_inventario       | SERIAL (es un INT)   | 4 bytes                     |
| stock_fisico         | INT                  | 4 bytes                     |
| stock_comprometido   | INT                  | 4 bytes                     |
| stock_minimo         | INT                  | 4 bytes                     |
| cod_producto         | VARCHAR(50)          | ≈ 15 bytes (promedio)       |
| cod_ubicacion        | VARCHAR(20)          | ≈ 15 bytes (promedio)       |

**Total (solo atributos):** ≈ 46 bytes x registro  
**Tamaño lógico con overhead:** 46 + 23 = ≈ 69 bytes x fila  

---

## Parámetros físicos (POSTGRES)
Asumiremos un total de **5,000 productos únicos** en el inventario para el cálculo inicial.

### CÁLCULOS:
- **Espacio Libre por Bloque:** (8192 - 24) × 0.8 = **6534.4 bytes por bloque**  
- **Cantidad de Filas por Bloque:** 6534.4 / 69 = **≈ 94 filas/bloque**  
- **Cantidad de Bloques:** 5,000 / 94 = **≈ 54 bloques**  
- **Tamaño Total:** 54 × 8192 = **442,368 bytes ≈ 0.44 MB**

---

## Parámetros

| Atributo                                  | Formato |
|-------------------------------------------|----------|
| Tamaño de bloque                          | 8 KB     |
| Tamaño de extent                          | —        |
| Estructura del registro y tamaño lógico   | ≈ 69 bytes x fila / ≈ 94 filas x bloque |
| Espacio libre reservado                   | `ALTER TABLE almacen.inventario SET (FILLFACTOR = 80);` |
| Tamaño total estimado                     | ≈ 0.44 MB |

# Diseño Físico: Tabla `movimiento`

## Estructura lógica (por registro)
Esta tabla es el historial de cada transacción que afecta el stock. Crecerá constantemente.

| Atributo            | Formato (PostgreSQL) | Tamaño                     |
|----------------------|----------------------|-----------------------------|
| cod_movimiento       | SERIAL (es un INT)   | 4 bytes                     |
| fecha_movimiento     | DATE                 | 4 bytes                     |
| hora_movimiento      | TIME                 | 8 bytes                     |
| tipo_movimiento      | VARCHAR(50)          | ≈ 25 bytes (promedio)       |
| cantidad             | INT                  | 4 bytes                     |
| cod_detalle_recepcion| INT                  | 4 bytes                     |
| cod_inventario       | INT                  | 4 bytes                     |

**Total (solo atributos):** ≈ 53 bytes x registro  
**Tamaño lógico con overhead:** 53 + 23 = ≈ 76 bytes x fila  

---

## Parámetros físicos (POSTGRES)
Asumiremos un historial inicial de **20,000 movimientos** para el cálculo.

### CÁLCULOS:
- **Espacio Libre por Bloque:** (8192 - 24) × 0.8 = **6534.4 bytes por bloque**  
- **Cantidad de Filas por Bloque:** 6534.4 / 76 = **≈ 85 filas/bloque**  
- **Cantidad de Bloques:** 20,000 / 85 = **≈ 236 bloques**  
- **Tamaño Total:** 236 × 8192 = **1,933,312 bytes ≈ 1.93 MB**

---

## Parámetros

| Atributo                                  | Formato |
|-------------------------------------------|----------|
| Tamaño de bloque                          | 8 KB     |
| Tamaño de extent                          | —        |
| Estructura del registro y tamaño lógico   | ≈ 76 bytes x fila / ≈ 85 filas x bloque |
| Espacio libre reservado                   | `ALTER TABLE almacen.movimiento SET (FILLFACTOR = 80);` |
| Tamaño total estimado                     | ≈ 1.93 MB |
