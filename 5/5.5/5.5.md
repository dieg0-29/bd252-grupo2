# 🧾 Tabla: VENTA

**Descripción:** Representa el registro central de cada operación comercial realizada.
**Propósito:** Consolidar los datos, estado, montos e identificación de cliente y vendedor para fines de control, reporte y auditoría.

**Reglas de negocio relevantes:**

* Cada venta tiene un identificador único.
* Una venta pertenece a un cliente y un vendedor.
* El total de la venta debe ser igual a la suma de sus productos menos descuentos más impuestos.
* Si la venta es anulada, no debe aceptar nuevos pagos.

| Clave y/o Restricción | Descripción                 |
| --------------------- | --------------------------- |
| **PK**                | COD_VENTA                   |
| **FK1**               | COD_CLIENTE → CLIENTE       |
| **FK2**               | COD_VENDEDOR → VENDEDOR     |
| **FK3**               | COD_TIPO_VENTA → TIPO_VENTA |
| **FK4**               | COD_CAJA → CAJA             |

| Columna          | Descripción                     | Propósito                     | Tipo de dato en el SGBD | Obligatoriedad | Unicidad | Restricciones                                               |
| ---------------- | ------------------------------- | ----------------------------- | ----------------------- | -------------- | -------- | ----------------------------------------------------------- |
| COD_VENTA        | Identificador único de la venta | Clave primaria y trazabilidad | VARCHAR(10)             | ✅              | ✅        | PK                                                          |
| FECHA_HORA_VENTA | Momento de la operación         | Orden temporal y auditoría    | TIMESTAMP               | ✅              | ❌        | DEFAULT CURRENT_TIMESTAMP                                   |
| ESTADO_VENTA     | Estado actual de la venta       | Seguimiento y control         | VARCHAR(20)             | ✅              | ❌        | CHECK (ESTADO_VENTA IN ('cancelada','por pagar','anulada')) |
| TIPO_VENTA       | Modalidad de la venta           | Segmentación y reporte        | VARCHAR(20)             | ✅              | ❌        | CHECK (TIPO_VENTA IN ('contado','crédito'))                 |
| DESCUENTO        | Descuento aplicado              | Cálculo total                 | DECIMAL(10,2)           | ❌              | ❌        | DEFAULT 0                                                   |
| IGV              | Impuesto aplicado               | Cálculo total                 | DECIMAL(10,2)           | ✅              | ❌        | DEFAULT 0                                                   |
| TOTAL            | Importe total de la venta       | Contabilidad y reportes       | DECIMAL(10,2)           | ✅              | ❌        | CHECK (TOTAL >= 0)                                          |

**Descripción:** Registra los productos vendidos dentro de una venta.
**Propósito:** Desglosar las cantidades, precios y valores totales de cada producto asociado a una venta.

**Reglas de negocio relevantes:**

* Cada detalle pertenece a una venta y un producto válidos.
* El total de la venta resulta de la suma de todos sus detalles.

# 🧾 Tabla: CAMBIO_PRODUCTO

**Descripción:** Sustitución de un producto vendido por otro dentro de una venta.
**Propósito:** Controlar cambios, movimientos de stock y dejar trazabilidad (venta, reclamo, productos de origen/destino y motivos).

**Reglas de negocio relevantes:**

* Debe referenciar **una venta** y (si corresponde) el **reclamo** que origina el cambio.
* Debe registrar **producto retornado** y **producto entregado** (no pueden ser iguales).
* Cada reclamo puede generar **a lo sumo un** registro de cambio (si tu política lo exige).

| Clave y/o Restricción | Descripción                                               |
| --------------------- | --------------------------------------------------------- |
| **PK**                | (COD_VENTA, COD_RECLAMO)                                  |
| **FK1**               | COD_VENTA → VENTA                                         |
| **FK2**               | COD_RECLAMO → RECLAMO                                     |
| **FK3**               | PRODUCTO_RETORNADO → PRODUCTO                             |
| **FK4**               | PRODUCTO_ENTREGADO → PRODUCTO                             |
| **CHK1**              | PRODUCTO_RETORNADO <> PRODUCTO_ENTREGADO                  |

| Columna            | Descripción                        | Propósito             | Tipo de dato en el SGBD | Obligatoriedad | Unicidad | Restricciones             |
| ------------------ | ---------------------------------- | --------------------- | ----------------------- | -------------- | -------- | ------------------------- |
| COD_VENTA          | Venta asociada al cambio           | Integridad            | VARCHAR(10)             | ✅              | ❌        | FK (VENTA)                |
| COD_RECLAMO        | Reclamo origen del cambio          | Trazabilidad          | VARCHAR(10)             | ✅              | ❌        | FK (RECLAMO)              |
| FECHA_HORA_CAMBIO  | Momento del cambio                 | Auditoría             | TIMESTAMP               | ✅              | ❌        | DEFAULT CURRENT_TIMESTAMP |
| PRODUCTO_RETORNADO | Producto devuelto por el cliente   | Ajuste de stock       | VARCHAR(10)             | ✅              | ❌        | FK (PRODUCTO)             |
| PRODUCTO_ENTREGADO | Producto que se entrega al cliente | Ajuste de stock       | VARCHAR(10)             | ✅              | ❌        | FK (PRODUCTO)             |
| MOTIVO_CAMBIO      | Razón del cambio                   | Clasificación/gestión | VARCHAR(100)            | ✅              | ❌        |                           |
| DESCRIPCION        | Detalle adicional del caso         | Contexto              | TEXT                    | ❌              | ❌        |                           |
