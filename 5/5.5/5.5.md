# 5.1.Diseño Lógico: Módulo de Ventas
## 💡 Esquema Relacional: Módulo de Clientes   

![Diagrama](<Diagrama Relacional (VENTAS).png>)

---

## Entidades:

| Tabla                  | Definición semántica                                                         |
| ---------------------- | ---------------------------------------------------------------------------- |
| **CLIENTE**            | Personas o empresas que compran; base para historial, beneficios y reclamos. |
| **VENDEDOR**           | Colaboradores que registran ventas y operan caja por turno.                  |
| **PRODUCTO**           | Ítems comercializables con precio, stock y categoría.                        |
| **VENTA**              | Transacciones que agrupan ítems, condiciones comerciales e impuestos.        |
| **PRODUCTO_VENTA**     | Detalle/ítems de una venta (tabla puente VENTA–PRODUCTO).                    |
| **PAGO**               | Cobros realizados a una venta (totales o parciales).                         |
| **METODO_PAGO**        | Catálogo de medios de pago disponibles.                                      |
| **COMPROBANTE**        | Documentos fiscales emitidos por pago.                                       |
| **TIPO_COMPROBANTE**   | Catálogo del tipo de comprobante (boleta, factura, etc.).                    |
| **TIPO_VENTA**         | Catálogo de modalidad de venta (contado, crédito, etc.).                     |
| **ESTADO_VENTA**       | Catálogo del estado del ciclo de la venta.                                   |
| **CAJA**               | Sesiones operativas de caja (apertura/cierre, totales).                      |
| **RECLAMO**            | Registros formales de disconformidad sobre una venta.                        |
| **DEVOLUCION**         | Restituciones de dinero derivadas de reclamos.                               |
| **MOTIVO_DEVOLUCION**  | Catálogo de causas para devoluciones.                                        |
| **ANULACION**          | Invalidaciones formales de ventas/pagos vinculadas a reclamos.               |
| **MOTIVO_ANULACION**   | Catálogo de causas para anulaciones.                                         |
| **CAMBIO_PRODUCTO**    | Intercambios de productos derivados de reclamos.                             |
| **MOTIVO_CAMBIO_PROD** | Catálogo de causas para cambios de producto.                                 |


---

## Atributos:

### **CLIENTE**

| Atributo                  | Semántica                       | Ontología (razón de ser)                 | Formato                  | Rango / Reglas                  |
| ------------------------- | ------------------------------- | ---------------------------------------- | ------------------------ | ------------------------------- |
| **cod_cliente**           | Identificador único de cliente. | Llave para relacionar ventas y reclamos. | `INT` o `VARCHAR(20)` | Único, no nulo                 |
| **fecha_ingreso_cliente** | Alta del cliente.               | Cohortes, antigüedad, auditoría.         | `DATETIME`              | ≤ ahora                        |
| **total_compras_cliente** | Total de compras realizadas        | KPI de valor del cliente.    | `INT`          | ≥ 0 |
| **total_puntos_cliente**  | Puntos de fidelización.         | Beneficios/descuentos.                   | `INT`                    | ≥ 0                            |

### **VENDEDOR**

| Atributo                   | Semántica                   | Ontología                     | Formato                  | Rango / Reglas      |
| -------------------------- | --------------------------- | ----------------------------- | ------------------------ | ------------------- |
| **cod_vendedor**           | Identificador del vendedor. | Atribución de ventas/caja.    | `INT` o `VARCHAR(20)` | Único, no nulo     |
| **fecha_ingreso_vendedor** | Fecha de ingreso.           | Gestión de personal/permisos. | `DATE`                   | ≤ hoy              |
| **total_ventas_vendedor**  | Total de ventas realizadas             | KPI de desempeño.             | `INT`          | ≥ 0 |

### **PRODUCTO**

| Atributo               | Semántica                   | Ontología                                 | Formato                  | Rango / Reglas               |
| ---------------------- | --------------------------- | ----------------------------------------- | ------------------------ | ---------------------------- |
| **cod_producto**       | Identificador del producto. | Llave del catálogo.                       | `INT` o `VARCHAR(20)` | Único                       |
| **nombre_producto**    | Denominación comercial.     | Visualización en comprobantes y listados. | `VARCHAR(120)`           | No vacío                    |
| **descp_producto**     | Descripción del producto.   | Ficha técnica/comercial.                  | `VARCHAR(500)`           | Opcional                    |
| **precio_producto**    | Precio de venta por unidad. | Cálculo de importes/impuestos.            | `DECIMAL(12,2)`          | ≥ 0.00                      |
| **stock_producto**     | Cantidad disponible.        | Control de inventario.                    | `INT`                    | ≥ 0                         |
| **categoria_producto** | Categoría o familia.        | Segmentación, reglas de precio/IGV.       | `VARCHAR(80)` o `ENUM`   | De acuerdo a catálogo |

### **VENTA**

| Atributo                  | Semántica                      | Ontología                                | Formato             | Rango / Reglas                    |
| ------------------------- | ------------------------------ | ---------------------------------------- | ------------------- | --------------------------------- |
| **cod_venta**             | Identificador de la venta.     | Trazabilidad de la transacción.          | `INT`            | Único                            |
| **fecha_hora_venta**      | Registro temporal de venta.    | Orden, cortes y auditoría.               | `DATETIME`         | ≤ ahora                          |
| **igv**                   | Monto de IGV de la venta.      | Cumplimiento tributario.                 | `DECIMAL(12,2)`     | ≥ 0; en PE ≈ 18% del monto   |
| **monto_venta**           | Importe total de la venta.     | Base para pagos/comprobantes.            | `DECIMAL(12,2)`     | ≥ 0; consistente con ítems       |
| **cod_tipo_venta (FK)**   | Tipo de venta.                 | Reglas (contado/crédito) y vencimientos. | `SMALLINT` | Debe existir en **TIPO_VENTA**   |
| **cod_estado_venta (FK)** | Estado del ciclo.              | Flujo de proceso/reportes.               | `SMALLINT`          | Debe existir en **ESTADO_VENTA** |
| **fecha_entrega**         | Fecha comprometida de entrega. | Logística/SLA.                           | `DATE`              | ≥ fecha_venta; opcional          |
| **cod_vendedor (FK)**     | Vendedor responsable.          | Trazabilidad/atribución.                 | `INT`            | Debe existir en **VENDEDOR**     |
| **cod_cliente (FK)**      | Cliente comprador.             | Historial y beneficios.                  | `INT`            | Debe existir en **CLIENTE**      |

### **PRODUCTO_VENTA**

| Atributo                                                                          | Semántica                         | Ontología                       | Formato         | Rango / Reglas                     |
| --------------------------------------------------------------------------------- | --------------------------------- | ------------------------------- | --------------- | ---------------------------------- |
| **cod_venta (FK)**                                                                | Venta a la que pertenece el ítem. | Agrupar detalle      | `INT`        | Debe existir en **VENTA**         |
| **cod_producto (FK)**                                                             | Producto vendido.                 | Identificar ítem y precio      | `INT`        | Debe existir en **PRODUCTO**      |
| **cantidad_producto**                                                             | Unidades del producto.            | Valor de ítem y stock          | `INT`           | **≥ 1**; típ. ≤ 9,999             |
| *sugeridos* **precio_unitario_al_momento**, **descuento_item**, **subtotal_item** | Congelan condiciones del ítem.    | Auditoría de precio/descuentos. | `DECIMAL(12,2)` | Coherentes con reglas comerciales |

### **PAGO**

| Atributo                   | Semántica                         | Ontología                              | Formato         | Rango / Reglas                   |
| -------------------------- | --------------------------------- | -------------------------------------- | --------------- | -------------------------------- |
| **nro_pago**               | Identificador/secuencia del pago. | Conciliación y trazabilidad del cobro. | `INT`        | Único por venta     |
| **cod_venta (FK)**         | Venta a la que aplica el pago.    | Disminuir saldo de la venta.           | `INT`        | Debe existir en **VENTA**       |
| **monto_pago**             | Importe pagado.                   | Cancelación total/parcial.             | `DECIMAL(12,2)` | > 0; ≤ saldo                    |
| **fecha_vencimiento_pago** | Límite para pagar (si crédito).   | Motor de cobranza/notificaciones.      | `DATE`          | Nulo si contado; ≥ fecha_venta  |
| **fecha_pago**             | Momento del cobro.                | Corte de caja/tesorería.               | `DATETIME`     | ≤ ahora                         |
| **cod_caja (FK)**          | Caja que registra el cobro.       | Arqueo/cuadre por turno.               | `INT`        | Debe existir en **CAJA**        |
| **nro_comprobante (FK)**   | Comprobante emitido.              | Sustento fiscal.                       | `VARCHAR(20)`   | Debe existir en **COMPROBANTE** |
| **cod_metodo_pago (FK)**   | Medio de pago.                    | Reglas de conciliación/costos.         | `SMALLINT`      | Debe existir en **METODO_PAGO** |

### **METODO_PAGO**

### **CAJA**

| Atributo                   | Semántica                        | Ontología                  | Formato         | Rango / Reglas          |
| -------------------------- | -------------------------------- | -------------------------- | --------------- | ----------------------- |
