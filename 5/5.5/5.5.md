# 5.1.Diseño Lógico: Módulo de Ventas
## 💡 Esquema Relacional: Módulo de Clientes   

![Diagrama](<Ventas - Esquema Relacional.jpeg>)

---
### **Nota:** Las tablas de ´producto' y 'cliente' pertenecen a otros módulos y han sido consideradas con los atributos mínimos para no perder la lógica del presente módulo

---

### 🧾 Tabla: `producto`

| Atributo        | Semántica                     | Ontología (razón de ser)                 | Formato      | Restricciones             |
| --------------- | ----------------------------- | ---------------------------------------- | ------------ | ------------------------- |
| cod_producto    | Código interno del producto   | Identificador único del catálogo         | INT          | PRIMARY KEY, autogenerado |
| nombre_producto | Nombre comercial del producto | Distingue y normaliza cada ítem vendible | VARCHAR(120) | NOT NULL, UNIQUE          |

---

### 🧾 Tabla: `cliente`

| Atributo       | Semántica                         | Ontología (razón de ser)                        | Formato      | Restricciones             |
| -------------- | --------------------------------- | ----------------------------------------------- | ------------ | ------------------------- |
| cod_cliente    | Código interno del cliente        | Identificador único del cliente                 | INT          | PRIMARY KEY, autogenerado |
| nombre_cliente | Razón social / nombre del cliente | Permite referenciar y evitar duplicados lógicos | VARCHAR(120) | NOT NULL, UNIQUE          |

---

### 🧾 Tabla: `metodo_pago`

| Atributo           | Semántica                 | Ontología (razón de ser)                                    | Formato     | Restricciones             |
| ------------------ | ------------------------- | ----------------------------------------------------------- | ----------- | ------------------------- |
| cod_metodo_pago    | Código del método         | Identificador único del medio de pago                       | INT         | PRIMARY KEY, autogenerado |
| nombre_metodo_pago | Nombre del método de pago | Normaliza opciones (efectivo, tarjeta, transferencia, etc.) | VARCHAR(60) | NOT NULL, UNIQUE          |

---

### 🧾 Tabla: `vendedor`

| Atributo               | Semántica                     | Ontología (razón de ser)                      | Formato | Restricciones             |
| ---------------------- | ----------------------------- | --------------------------------------------- | ------- | ------------------------- |
| cod_vendedor           | Código del vendedor           | Identificador único del colaborador de ventas | INT     | PRIMARY KEY, autogenerado |
| fecha_ingreso_vendedor | Fecha de alta del vendedor    | Control de antigüedad y trazabilidad          | DATE    | NOT NULL, DEFAULT NOW()   |
| total_ventas_vendedor  | Contador de ventas realizadas | Métrica acumulada para gestión comercial      | INT     | DEFAULT 0                 |

---

### 🧾 Tabla: `tipo_venta`

| Atributo         | Semántica                      | Ontología (razón de ser)                                               | Formato     | Restricciones             |
| ---------------- | ------------------------------ | ---------------------------------------------------------------------- | ----------- | ------------------------- |
| cod_tipo_venta   | Código único del tipo de venta | Identificador interno del tipo de transacción (contado, crédito, etc.) | INT         | PRIMARY KEY, autogenerado |
| descp_tipo_venta | Descripción del tipo de venta  | Define la modalidad comercial de la transacción                        | VARCHAR(20) | NOT NULL, UNIQUE          |

---

### 🧾 Tabla: `estado_venta`

| Atributo           | Semántica                        | Ontología (razón de ser)                             | Formato     | Restricciones             |
| ------------------ | -------------------------------- | ---------------------------------------------------- | ----------- | ------------------------- |
| cod_estado_venta   | Código único del estado de venta | Identificador del estado actual del proceso de venta | INT         | PRIMARY KEY, autogenerado |
| descp_estado_venta | Descripción del estado de venta  | Indica si la venta está activa, anulada o finalizada | VARCHAR(20) | NOT NULL, UNIQUE          |

---

### 🧾 Tabla: `tipo_comprobante`

| Atributo               | Semántica                           | Ontología (razón de ser)                                | Formato     | Restricciones             |
| ---------------------- | ----------------------------------- | ------------------------------------------------------- | ----------- | ------------------------- |
| cod_tipo_comprobante   | Código del tipo de comprobante      | Identifica el tipo de documento fiscal emitido          | INT         | PRIMARY KEY, autogenerado |
| descp_tipo_comprobante | Descripción del tipo de comprobante | Especifica si es factura, boleta, nota de crédito, etc. | VARCHAR(40) | NOT NULL, UNIQUE          |

---

### 🧾 Tabla: `estado_pago`

| Atributo           | Semántica                 | Ontología (razón de ser)                               | Formato     | Restricciones             |
| ------------------ | ------------------------- | ------------------------------------------------------ | ----------- | ------------------------- |
| cod_estado_pago    | Código del estado de pago | Identificador del estado del pago (pendiente o pagado) | INT         | PRIMARY KEY, autogenerado |
| nombre_estado_pago | Nombre del estado de pago | Define el estado actual del pago de una venta          | VARCHAR(20) | NOT NULL, UNIQUE          |

---

### 🧾 Tabla: `venta`

| Atributo         | Semántica                     | Ontología (razón de ser)                     | Formato       | Restricciones             |
| ---------------- | ----------------------------- | -------------------------------------------- | ------------- | ------------------------- |
| cod_venta        | Código de venta               | Identificador único de la venta realizada    | INT           | PRIMARY KEY, autogenerado |
| fecha_hora_venta | Fecha y hora de venta         | Momento exacto en que se registró la venta   | TIMESTAMP     | NOT NULL, DEFAULT NOW()   |
| monto_venta      | Monto total de la venta       | Valor económico de la transacción            | NUMERIC(12,2) | NOT NULL, > 0             |
| igv              | Impuesto General a las Ventas | Impuesto aplicado sobre el monto de la venta | NUMERIC(12,2) | NOT NULL, > 0             |
| dscto_venta      | Descuento aplicado            | Reducción monetaria aplicada a la venta      | NUMERIC(12,2) | DEFAULT 0                 |
| fecha_venta      | Fecha de emisión              | Día de la venta                              | DATE          | NOT NULL, DEFAULT NOW()   |
| fecha_entrega    | Fecha de entrega del producto | Día comprometido para entrega al cliente     | DATE          | NOT NULL, ≥ fecha_venta   |
| cod_tipo_venta   | Tipo de venta                 | Relación con `tipo_venta`                    | INT           | FK NOT NULL               |
| cod_estado_venta | Estado de venta               | Relación con `estado_venta`                  | INT           | FK NOT NULL               |
| cod_vendedor     | Vendedor responsable          | Referencia a tabla `vendedor`                | INT           | FK NOT NULL               |
| cod_cliente      | Cliente asociado              | Referencia a tabla `cliente`                 | INT           | FK NOT NULL               |

---

### 🧾 Tabla: `producto_venta`

| Atributo          | Semántica                | Ontología (razón de ser)       | Formato | Restricciones                         |
| ----------------- | ------------------------ | ------------------------------ | ------- | ------------------------------------- |
| cod_venta         | Código de venta          | Identifica la venta asociada   | INT     | FK NOT NULL                           |
| cod_producto      | Código del producto      | Identifica el producto vendido | INT     | FK NOT NULL                           |
| cantidad_producto | Cantidad vendida         | Número de unidades vendidas    | INT     | NOT NULL, > 0                         |
| (PK compuesta)    | Clave primaria compuesta | Unicidad de producto por venta | —       | PRIMARY KEY (cod_venta, cod_producto) |

| fecha_emision        | Fecha de emisión       | Día en que se generó el comprobante   | DATE    | NOT NULL                  |

---

### 🧾 Tabla: `caja`

| Atributo            | Semántica                  | Ontología (razón de ser)           | Formato       | Restricciones             |
| ------------------- | -------------------------- | ---------------------------------- | ------------- | ------------------------- |
| cod_caja            | Código de caja             | Identificador de sesión de caja    | INT           | PRIMARY KEY, autogenerado |

| fecha_hora_cierre   | Fecha/hora de cierre       | Fin del turno de caja              | TIMESTAMP     | NULL                      |

| monto_apertura      | Monto inicial              | Saldo inicial de la caja           | NUMERIC(12,2) | DEFAULT 0                 |

| cantidad_ventas     | Número de ventas           | Cantidad de operaciones procesadas | INT           | DEFAULT 0                 |
