## Entidades:

| Entidad        | Descripción semántica                                                                                  | Clave primaria | 
|----------------|--------------------------------------------------------------------------------------------------------|----------------|
| PERSONA        | Representa a una persona natural usada como base para clientes y vendedores.                           | documento_identidad | 
| NOMBRE         | Fragmenta el nombre de una persona para normalizar (nombres y apellidos).                              | id_nombre      | 
| DIRECCION      | Dirección física asociada a una persona (ubicación).                                                   | id_direccion   | 
| CONTACTO       | Medios de contacto de una persona.                                                                     | id_contacto    | 
| CLIENTE        | Persona que compra a la ferretería y acumula compras/puntos.                                           | id_cliente     | 
| VENDEDOR       | Persona que vende/atiende y a quien se puede medir por ventas.                                         | id_vendedor    | 
| PRODUCTO       | Artículo comercializable.                                                                              | id_producto    | 
| VENTA          | Transacción de venta realizada a un cliente por un vendedor.                                           | id_venta       | 
| PRODUCTO_VENTA | Intersección VENTA–PRODUCTO que almacena líneas de detalle.                                            | (id_venta, id_producto) | 
| PAGO           | Registro de un pago (total o parcial) asociado a una venta.                                            | num_pago       | 
| CAJA           | Turno de caja que agrupa ventas/pagos y permite cuadrar dinero.                                        | id_caja        | 
| COMPROBANTE    | Documento tributario generado a partir de pagos/ventas (boleta, factura, notas).                       | nro_comprobante| 
| METODO_PAGO    | Catálogo de métodos de pago aceptados.                                                                 | id_metodo      | 

---

## Atributos:

| Atributo            | Formato / Tipo   | Posibles Valores            | Semántica                             | Ontología  |
|----------------------|------------------|-------------------------------------------------|----------------------------------------------------------|--------------------------|
| apellido_mat         | VARCHAR(50)      | Letras, opcional                                | Apellido materno de una persona                          | Identificar de forma más precisa a la persona en documentos oficiales. |
| apellido_pat         | VARCHAR(50)      | Letras, obligatorio                             | Apellido paterno de una persona                          | Garantizar la identificación principal de la persona según costumbre legal. |
| cantidad             | INT              | > 0                                             | Número de unidades de un producto en una venta            | Registrar el detalle de cantidades vendidas por producto. |
| cantidad_ventas      | INT              | ≥ 0                                             | Número de ventas procesadas en una caja                  | Medir el volumen de operaciones realizadas en un turno de caja. |
| ciudad               | VARCHAR(50)      | Catálogo de ciudades                            | Ciudad asociada a una dirección                          | Permitir ubicar geográficamente a clientes y vendedores. |
| descripcion_metodo   | VARCHAR(50)      | EFECTIVO, TARJETA, TRANSFERENCIA, YAPE/PLIN     | Nombre del método de pago                                | Mostrar al usuario la forma en que se puede realizar el pago. |
| descripcion_producto | VARCHAR(200)     | Texto libre                                     | Características del producto                             | Ayudar a detallar y diferenciar productos similares. |
| distrito             | VARCHAR(50)      | Catálogo de distritos                           | Distrito asociado a la dirección                         | Mayor precisión en la localización geográfica. |
| documento_identidad  | VARCHAR(20)      | DNI (8), RUC (11), CE/Pasaporte                 | Identificador único de una persona                       | Diferenciar de forma inequívoca a clientes y vendedores en el sistema. |
| estado_venta         | VARCHAR(20)      | PENDIENTE, PARCIAL, PAGADA, VENCIDA, ANULADA    | Situación actual de una venta                            | Controlar el ciclo de vida de cada transacción. |
| fecha_hora_apertura  | DATETIME         | ≥ fecha actual                                  | Momento en que se abre la caja                           | Registrar el inicio de operaciones diarias. |
| fecha_hora_cierre    | DATETIME         | ≥ fecha_hora_apertura                           | Momento en que se cierra la caja                         | Determinar fin de operaciones y controlar el cuadre. |
| fecha_hora_venta     | DATETIME         | ≥ apertura de caja                              | Momento exacto de una venta                              | Garantizar la trazabilidad temporal de las transacciones. |
| fecha_ingreso        | DATE             | Fecha válida                                    | Fecha de contratación de un vendedor                     | Conocer la antigüedad laboral del personal de ventas. |
| fecha_pago           | DATE             | ≥ fecha de venta                                | Día en que se registró un pago                           | Asociar pagos a ventas para el control de cuentas por cobrar/pagar. |
| fecha_vencimiento    | DATE             | ≥ fecha de venta                                | Fecha límite de pago en ventas a crédito                 | Establecer plazos para gestionar cobranzas. |
| id_caja              | INT (PK)         | Autonumérico                                    | Identificador único de una caja                          | Diferenciar cada sesión/turno de caja en el sistema. |
| id_cliente           | INT (PK)         | Autonumérico                                    | Identificador único de un cliente                        | Referenciar a cada cliente en ventas y pagos. |
| id_contacto          | INT (PK)         | Autonumérico                                    | Identificador de un medio de contacto                    | Registrar múltiples formas de comunicación de una persona. |
| id_direccion         | INT (PK)         | Autonumérico                                    | Identificador único de dirección                         | Permitir almacenar distintas direcciones por persona. |
| id_metodo            | INT (PK)         | Autonumérico                                    | Identificador único del método de pago                   | Vincular los pagos a un catálogo estandarizado. |
| id_nombre            | INT (PK)         | Autonumérico                                    | Identificador de un registro de nombres                  | Normalizar la información de identidad personal. |
| id_producto          | INT (PK)         | Autonumérico                                    | Identificador único de producto                          | Referenciar artículos en ventas y pagos. |
| id_venta             | INT (PK)         | Autonumérico                                    | Identificador único de una venta                         | Garantizar unicidad y trazabilidad de transacciones. |
| id_vendedor          | INT (PK)         | Autonumérico                                    | Identificador único de un vendedor                       | Asignar cada venta a su responsable. |
| monto_apertura       | DECIMAL(10,2)    | ≥ 0                                             | Dinero inicial declarado en caja                         | Servir de base para calcular descuadres y arqueo de caja. |
| monto_cierre         | DECIMAL(10,2)    | ≥ 0                                             | Dinero contado al cierre                                 | Verificar cuadratura de caja contra ventas registradas. |
| monto_pago           | DECIMAL(10,2)    | > 0                                             | Valor de un pago realizado                               | Determinar el importe abonado en cada transacción. |
| monto_venta          | DECIMAL(10,2)    | > 0                                             | Valor total de la venta                                  | Calcular ingresos y reportes contables. |
| monto_ventas_total   | DECIMAL(10,2)    | ≥ 0                                             | Suma de todas las ventas en la caja                      | Controlar los ingresos por turno de caja. |
| nombre_producto      | VARCHAR(100)     | Texto libre                                     | Denominación comercial del producto                      | Permitir identificar rápidamente el artículo en catálogo y boletas. |
| nombre_via           | VARCHAR(100)     | Texto libre                                     | Nombre de la calle/avenida                               | Ubicar de forma detallada la dirección de una persona. |
| nombres              | VARCHAR(100)     | Texto libre                                     | Nombres de la persona                                    | Identificar de manera clara al cliente/vendedor. |
| nro_comprobante      | VARCHAR(20) (PK) | Serie-correlativo (ej: F001-00012345)           | Número único de documento tributario                     | Garantizar validez fiscal y control de SUNAT. |
| num_pago             | INT (PK)         | Autonumérico                                    | Identificador único de un pago                           | Distinguir cada operación de pago en el sistema. |
| numero               | VARCHAR(10)      | Números/letras                                  | Número del inmueble en la dirección                      | Identificar la ubicación precisa dentro de una vía. |
| precio_unitario      | DECIMAL(10,2)    | > 0                                             | Precio por unidad del producto                           | Determinar el valor comercial básico de los productos. |
| tipo_comprobante     | VARCHAR(20)      | BOLETA, FACTURA, NCREDITO, NDEBITO              | Clasificación del comprobante                            | Indicar el marco legal/fiscal bajo el que se emite el documento. |
| tipo_contacto        | VARCHAR(20)      | EMAIL, TELÉFONO, CELULAR, WHATSAPP              | Tipo de medio de contacto                                | Clasificar los canales de comunicación con clientes. |
| tipo_pago            | VARCHAR(20)      | CONTADO, CRÉDITO                                | Modalidad de la transacción financiera                   | Determinar condiciones de pago y riesgos asociados. |
| total_compras        | INT              | ≥ 0                                             | Cantidad acumulada de compras realizadas                 | Permitir segmentar clientes según su historial. |
| total_puntos         | INT              | ≥ 0                                             | Puntos acumulados como beneficios del cliente            | Implementar programas de fidelización y promociones. |
| total_ventas         | INT              | ≥ 0                                             | Ventas totales realizadas por un vendedor                | Medir desempeño y productividad del personal. |
| valor_contacto       | VARCHAR(100)     | Email, número telefónico, usuario de mensajería | Información concreta del medio de contacto               | Permitir comunicación directa con clientes y vendedores. |
| vendedor_apertura    | INT (FK)         | Debe existir en VENDEDOR                        | Vendedor que abrió la caja                               | Registrar la responsabilidad de inicio del turno de caja. |
| vendedor_cierre      | INT (FK)         | Debe existir en VENDEDOR                        | Vendedor que cerró la caja                               | Registrar la responsabilidad del cierre de caja. |
| via                  | VARCHAR(50)      | Calle, Avenida, Jr., Psje., etc.                | Tipo de vía de la dirección                              | Establecer la clasificación formal de la ubicación. |
