# 5.4. Diseño Lógico: Módulo de Abastecimiento
## 💡 Esquema Relacional: Módulo de Abastecimiento

---

## Entidades:

| Entidad | Descripción semántica | Clave primaria |
| :--- | :--- | :--- |
| ÁREA | Representa un departamento o área física dentro de la empresa. | id_area |
| USUARIO | Representa a un empleado con acceso al sistema y rol definido. | id_usuario |
| SOLICITUD_ABASTECIMIENTO | Documento interno que formaliza el requerimiento de productos. | id_solicitud |
| PRODUCTO | Artículo comercializable que se gestiona en el inventario. | id_producto |
| RUBRO | Categoría comercial para clasificar productos y proveedores. | id_rubro |
| PROVEEDOR | Empresa o persona que suministra productos. | id_proveedor |
| COTIZACIÓN | Documento formal con la oferta de precios de un proveedor. | id_cotizacion |
| SOLICITUD_COTIZACION | Petición formal de precios a uno o varios proveedores. | id_solicitud |
| ORDEN_COMPRA | Documento que autoriza formalmente la adquisición de productos. | id_orden |
| MONITOREO_COMPRA | Registro del seguimiento de una orden de compra. | id_monitoreo |
| RECEPCIÓN | Registro de la llegada y entrada física de mercancía. | id_recepcion |
| RECLAMO | Documento que formaliza una queja sobre la mercancía recibida. | id_reclamo |
| SOLICITUD_ABASTECIMIENTO_PRODUCTO | Intersección entre una solicitud y sus productos. | (id_solicitud, id_producto) |
| RUBRO_PROVEEDOR | Intersección entre un rubro y los proveedores que lo atienden. | (id_rubro, id_proveedor) |
| SOLICITUD_COTIZACION_PRODUCTO | Intersección entre una solicitud de cotización y sus productos. | (id_solicitud, id_producto) |
| CONTACTO_PROVEEDOR | Atributo multivaluado para registrar múltiples contactos de un proveedor. | (id_proveedor, tipo_contacto) |
| TIPO_PRODUCTO | Atributo multivaluado para categorizar un producto. | (id_producto, codigo) |

---

## Atributos:

| Atributo | Formato / Tipo | Posibles Valores | Semántica | Ontología |
| :--- | :--- | :--- | :--- | :--- |
| id_area | VARCHAR(50) | Texto único | Identificador de un área. | Clave primaria para la entidad Área. |
| nombre | VARCHAR(100) | Texto | Nombre descriptivo del área o rubro. | Identificador legible del registro. |
| observaciones | TEXT | Texto libre | Notas o comentarios adicionales. | Documentar detalles relevantes. |
| estado | VARCHAR(50) | Texto | Situación actual de un registro. | Controlar el ciclo de vida de una entidad (ej. 'Activo', 'Inactivo'). |
| id_usuario | VARCHAR(50) | Texto único | Identificador único de un usuario. | Clave primaria para la entidad Usuario. |
| correo | VARCHAR(100) | Formato de email, único | Correo electrónico del usuario. | Servir como identificador de inicio de sesión. |
| rol | VARCHAR(50) | Enumeración | Función del usuario en la empresa. | Definir permisos y acceso al sistema. |
| hash_contrasena | VARCHAR(255) | Texto encriptado | Contraseña del usuario. | Proteger el acceso al sistema. |
| fecha_creacion | DATE | Fecha | Momento en que se creó el registro. | Trazabilidad del registro. |
| id_area | VARCHAR(50) | Debe existir en Área | Identificador del área a la que pertenece el usuario. | Clave foránea que relaciona Usuario con Área. |
| id_solicitud | VARCHAR(50) | Texto único | Identificador de una solicitud de abastecimiento. | Clave primaria para Solicitud de Abastecimiento. |
| fecha | DATE | Fecha | La fecha en que se creó una solicitud, recepción o reclamo. | Trazabilidad temporal de las transacciones. |
| hora | TIME | Hora | La hora exacta en que se creó un registro. | Complementar la fecha para un registro preciso. |
| estado | VARCHAR(50) | Enumeración | Situación de una solicitud. | Controlar el flujo de trabajo (ej. 'Pendiente', 'Aprobada'). |
| fecha_requerida | DATE | Fecha | Fecha en la que se necesitan los productos. | Establecer plazos logísticos. |
| id_usuario_solicitante | VARCHAR(50) | Debe existir en Usuario | Usuario que emitió la solicitud. | Clave foránea que rastrea la autoría de la solicitud. |
| id_producto | VARCHAR(50) | Texto único | Identificador de un producto. | Clave primaria para Producto. |
| precio | DECIMAL(10, 2) | Mayor a 0 | Precio de venta del producto. | Valor comercial del artículo. |
| unidad_medida | VARCHAR(50) | Enumeración | Unidad de conteo para el producto. | Estandarizar el inventario. |
| stock_actual | INT | Mayor o igual a 0 | Cantidad actual de un producto. | Controlar el inventario disponible. |
| stock_minimo | INT | Mayor a 0 | Cantidad mínima para alerta de reposición. | Servir como umbral de alerta. |
| ubicacion | VARCHAR(100) | Texto | Ubicación física del producto en el almacén. | Facilitar la búsqueda del producto. |
| id_rubro | VARCHAR(50) | Texto único | Identificador de un rubro comercial. | Clave primaria para la entidad Rubro. |
| id_proveedor | VARCHAR(50) | Texto único | Identificador de un proveedor. | Clave primaria para la entidad Proveedor. |
| nombre_comercial | VARCHAR(100) | Texto | Nombre público o comercial del proveedor. | Identificación amigable para el usuario. |
| razon_social | VARCHAR(100) | Texto | Nombre legal del proveedor. | Fines contables y de facturación. |
| RUC | VARCHAR(20) | Texto único | Número de identificación fiscal del proveedor. | Identificar legalmente al proveedor. |
| direccion | VARCHAR(255) | Texto libre | Dirección física del proveedor. | Datos de contacto y logística. |
| id_cotizacion | VARCHAR(50) | Texto único | Identificador de una cotización. | Clave primaria para Cotización. |
| modalidad_pago | VARCHAR(100) | Enumeración | Términos de pago de la cotización. | Definir las condiciones financieras de la transacción. |
| fecha_emision | DATE | Fecha | Fecha en que se emitió una cotización o una orden de compra. | Trazabilidad temporal de los documentos. |
| fecha_garantia | DATE | Fecha | Fecha límite de validez de la cotización. | Gestionar la vigencia de la oferta. |
| monto_total | DECIMAL(10, 2) | Mayor a 0 | Valor total de la cotización. | Comparación de precios y presupuesto. |
| id_orden | VARCHAR(50) | Texto único | Identificador de una orden de compra. | Clave primaria para Orden de Compra. |
| monto | DECIMAL(10, 2) | Mayor a 0 | Costo total de una orden de compra. | Fines contables y de auditoría. |
| id_usuario_emisor | VARCHAR(50) | Debe existir en Usuario | Usuario que emitió la orden de compra. | Clave foránea para la autoría de la orden. |
| id_cotizacion | VARCHAR(50) | Debe existir en Cotización | Cotización que dio origen a la orden de compra. | Clave foránea para la trazabilidad de la compra. |
| id_monitoreo | VARCHAR(50) | Texto único | Identificador de un registro de monitoreo. | Clave primaria para Monitoreo de Compra. |
| fecha_entrega | DATE | Fecha | Fecha de entrega estimada. | Controlar el plazo de entrega. |
| hora_entrega | TIME | Hora | Hora de entrega estimada. | Complementar el registro de entrega. |
| fecha_real_entrega | DATE | Fecha | La fecha real en que se recibió el pedido. | Medir la eficiencia del proveedor. |
| id_recepcion | VARCHAR(50) | Texto único | Identificador de una recepción. | Clave primaria para Recepción. |
| observacion | TEXT | Texto libre | Comentarios sobre la recepción. | Documentar anomalías en la entrega. |
| id_usuario_receptor | VARCHAR(50) | Debe existir en Usuario | Usuario que recibió la mercancía. | Clave foránea para la responsabilidad de la recepción. |
| id_reclamo | VARCHAR(50) | Texto único | Identificador de un reclamo. | Clave primaria para Reclamo. |
| monto_devuelto | DECIMAL(10, 2) | Mayor o igual a 0 | Valor monetario de la compensación. | Fines contables y de resolución del reclamo. |
| id_recepcion | VARCHAR(50) | Debe existir en Recepción | Recepción que originó el reclamo. | Clave foránea para vincular el reclamo a una entrega. |
| cantidad_solicitada | INT | Mayor a 0 | Cantidad de un producto en una solicitud. | Detalle de las cantidades requeridas en cada línea de la solicitud. |
| fecha_asociacion | DATE | Fecha | Fecha en que el proveedor fue asociado a un rubro. | Trazabilidad histórica de la relación. |
| tipo_contacto | VARCHAR(50) | Enumeración | Tipo de contacto (ej. 'teléfono', 'correo'). | Clasificar los canales de comunicación. |
| valor_contacto | VARCHAR(100) | Texto | Número, correo o valor del contacto. | Dato concreto para la comunicación. |
| codigo | VARCHAR(50) | Texto | Código de clasificación del tipo de producto. | Identificador de la categoría. |
| nombre_del_tipo | VARCHAR(100) | Texto | Nombre de la categoría. | Identificación legible de la clasificación. |
