# 10.1.3. Módulo 3 / Integrante 3

# Indices

## Se da prioridad a tablas que presentarán gran cantidad de registros y que serán consultadas con regularidad en los flujos primarios (R-308, R-305, R-302, R-306).

### 1. MOVIMIENTO por cod_inventario y fecha_movimiento

Este es el índice **más importante** del módulo. Se utiliza directamente en el **R-308 (Reporte Kardex)**. La tabla `MOVIMIENTO` será la más grande, ya que registra cada entrada, salida y ajuste. La consulta principal siempre filtrará por un `cod_inventario` (para ver el historial de un solo producto) y ordenará por fecha.

```
--SIN INDICE
EXPLAIN ANALYZE
SELECT
  m.fecha_movimiento,
  m.hora_movimiento,
  m.tipo_movimiento,
  m.cantidad
FROM "FERRETERIA".movimiento m
WHERE
  m.cod_inventario = 123 -- (ID de un inventario de producto)
ORDER BY
  m.fecha_movimiento DESC, m.hora_movimiento DESC;

```

```
--CREAMOS INDICE
CREATE INDEX idx_movimiento_inventario_fecha
ON "FERRETERIA".movimiento(cod_inventario, fecha_movimiento DESC, hora_movimiento DESC);

--CON INDICE
EXPLAIN ANALYZE
SELECT
  m.fecha_movimiento,
  m.hora_movimiento,
  m.tipo_movimiento,
  m.cantidad
FROM "FERRETERIA".movimiento m
WHERE
  m.cod_inventario = 123 -- (ID de un inventario de producto)
ORDER BY
  m.fecha_movimiento DESC, m.hora_movimiento DESC;

```

### 2. INVENTARIO por cod_producto

Este índice es vital para la **R-305 (Consulta de Inventario)**, **R-304 (Control de Stock)** y como punto de partida para el **R-308 (Kardex)**. La tabla `INVENTARIO` es el "cerebro" del almacén que conecta productos con ubicaciones. Las búsquedas de usuario casi siempre comenzarán con un `cod_producto` para encontrar todas sus ubicaciones y stocks.

```
--SIN INDICE
EXPLAIN ANALYZE
SELECT
  i.stock_fisico,
  i.stock_comprometido,
  u.cod_ubicacion_calculado,
  u.tipo_ubicacion
FROM "FERRETERIA".inventario i
JOIN "FERRETERIA".ubicacion u ON i.cod_ubicacion = u.cod_ubicacion
WHERE
  i.cod_producto = 45; -- (ID de un producto, ej. 'Cemento')

```

```
--CREAMOS INDICE
CREATE INDEX idx_inventario_producto
ON "FERRETERIA".inventario(cod_producto);

--CON INDICE
EXPLAIN ANALYZE
SELECT
  i.stock_fisico,
  i.stock_comprometido,
  u.cod_ubicacion_calculado,
  u.tipo_ubicacion
FROM "FERRETERIA".inventario i
JOIN "FERRETERIA".ubicacion u ON i.cod_ubicacion = u.cod_ubicacion
WHERE
  i.cod_producto = 45; -- (ID de un producto, ej. 'Cemento')

```

### 3. OPERADOR_RESERVA_ALMACEN por cod_usuario

Este índice es clave para los flujos operativos **R-302 (Recepción)** y **R-306 (Picking)**. Se utiliza para construir la lista de tareas ("Bandeja de Tareas") asignadas a un operador específico. Sin este índice, el sistema tendría que escanear toda la tabla `operador_reserva_almacen` cada vez que un operario inicia sesión o refresca su panel.

```
--SIN INDICE
EXPLAIN ANALYZE
SELECT
  r.*,
  c.fecha_cupo,
  t.hora_inicio
FROM "FERRETERIA".reserva_almacen r
JOIN "FERRETERIA".operador_reserva_almacen ora ON r.cod_reserva = ora.cod_reserva
JOIN "FERRETERIA".cupo_disponible c ON r.cod_cupo = c.cod_cupo
JOIN "FERRETERIA".turno_almacen t ON c.cod_turno = t.cod_turno
WHERE
  ora.cod_usuario = 10 -- (ID del operador logueado)
  AND r.tipo_reserva = 'Recepcion'
  AND r.estado = 'Confirmado';

```

```
--CREAMOS INDICE
CREATE INDEX idx_op_reserva_usuario
ON "FERRETERIA".operador_reserva_almacen(cod_usuario);

--CON INDICE
EXPLAIN ANALYZE
SELECT
  r.*,
  c.fecha_cupo,
  t.hora_inicio
FROM "FERRETERIA".reserva_almacen r
JOIN "FERRETERIA".operador_reserva_almacen ora ON r.cod_reserva = ora.cod_reserva
JOIN "FERRETERIA".cupo_disponible c ON r.cod_cupo = c.cod_cupo
JOIN "FERRETERIA".turno_almacen t ON c.cod_turno = t.cod_turno
WHERE
  ora.cod_usuario = 10 -- (ID del operador logueado)
  AND r.tipo_reserva = 'Recepcion'
  AND r.estado = 'Confirmado';
```
